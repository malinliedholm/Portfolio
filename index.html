oVRseer

C#

Unity

Blender

oVRseer is an asymmetrical multiplayer prop hunt VR game developed in Unity as a part of a group project in the course DH2413 Advanced Graphics and Interaction at KTH. In the game one player plays in VR and takes the role of the Overseer, with the goal of finding and eliminating the other players. The other players play can play on either desktop or an android phone. These players play as Tiny golems and their goal is to collect keys that are hidden across the map. When enough keys are collected, doors open that the Tiny golem players can escape through. While trying to find the keys the Tiny golem players should avoid getting noticed by the Overseer. They can do this by for example morph into different props that are scattered across the map or by trying to impersonate the NPCs that are walking around the map. The game ends when for each of the Tiny golems, they have either managed to escape or been eliminated by the Overseer.

During the project I worked with implementing the key collecting system for the Tiny golem players, where C# was the main programming language. I also worked on implementing a simple shader to create a light beam effect that shows up in front of the doors when they are opened so that the Tiny golem players can easier find there way to the door. This was my first time writing a shader in Unity, so I learned the basics of writing shaders in HLSL and Shaderlab. Futhermore I worked with creating some of the 3D models for the game in Blender as well as constructing parts of the final map.

If you want to learn even more about the project you can check out the project website

Sabel & Tiny

C#

Unity

Sabel and Tiny is a 2D platform puzzle game developed in Unity as a part of a group project in the course DH2650 Computer Game Design at KTH.

During this project it was my first time working in Unity, so I learned the basics of working with the engine. My main contribution to the project was implementing the movement and switching of the characters. I also implemented the puzzle locks which are locks that prevent the characters from reaching different parts of the map. To unlock a lock and progress further in the game the player needs to find the corresponding puzzle piece that is located somewhere on the map. I also had the main responsibility of documenting the work with the game, such as information about gameplay and game mechanics, characters, and game world in a game design document.

If you want to learn even more about the project you can check out the project website

Ray tracer

C++

SDL

As a part of a project in the course DH2323 Computer Graphics and Interaction at KTH I implemented a basic ray tracer in C++ and SDL.

The ray tracer was an extension on one of the lab assignments I made in the same course. The implemented ray tracer can render diffuse and specular materials on triangulated meshes and spheres. The ray tracer also renders hard shadows. Refraction was also implemented as a part of the ray tracer to be able to render transparent materials.

If you want to learn even more about the project you can read the project report that is available here: Project report
